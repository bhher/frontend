## 팀 프로젝트 로드맵 요약

- **파운데이션**  
  목표 확정, 역할 분담, 기능 정의, 일정 계획 → *결과물: 프로젝트 제안서*

- **기획 & 디자인 (Figma)**  
  와이어프레임 → UI 디자인 → UX 흐름 → 피드백 반영 → 디자인 시스템 정리 → *결과물: Figma 링크, 스타일 가이드*

- **기술 스택 세팅 (React + Git)**  
  React 초기화, 폴더 구조, GitHub 리포, 브랜치 전략, 코드 컨벤션, 공용 패키지 → *결과물: Repo + README + 협업 규칙*

- **개발 단계**  
  주차별 목표 (공통 컴포넌트 → 라우팅 → API 연동 → 상태 관리 → 스타일링 → QA) → *결과물: `main` 브랜치 완성본*

- **테스트 & 디버깅**  
  기능 체크 리스트, 사용자 테스트, 에러 로그 점검, 리팩토링 → *결과물: v1.0 릴리즈 태그*

- **발표 준비**  
  슬라이드, 데모(영상/실시간), 발표 분담, 리허설, 핵심 메시지 정리 → *결과물: PPT / Demo 링크 / GitHub 링크 / 발표 대본*

- **최종 포트폴리오 구성**  
  `src/`·`design/`·`docs/`·`assets/`·`README`·`demo-video.mp4`

- **발표 시 강조 포인트**  
  협업 전략, UI/UX 완성도, 기술 구현, 문제 해결 사례, 팀워크 성장

- **핵심 체크 리스트**  
  공통 이해 확보(제안서·디자인 시스템), PR 기반 협업, 일정·테스트 관리, 발표 준비 철저
